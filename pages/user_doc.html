3-D NETWORK MAPS

Visualizing the evolution of human networks

Overview

Two csv files need to be uploaded in order to model human networks over time using this program. One file must contain information about the individuals and the other must contain information about the relationships. The program is designed to operate on files formatted with the standard described here, but with specific column names. The file referring to the attributes of the individuals will be referred to as the entity file and the file referring to relationships will be referred to as the relationship file. The following are the requirements for the column names and a brief description of each column. All dates should written in the ISO 8601 format but the program will attempt to parse any date format inputted.

Entity File Requirements

The following are the column headers and what information they should contain for the program to parse the entity file.

  • id

    • A unique id for each entity

  • name

    • The name of the entity that will be displayed in the model

  • dob

    • The date of birth of the entity

  • active

    • The date of death of the entity or last date entity was active

Relationship File Requirements

The following are the column headers and what information they should contain for the program to parse the relationship file.

  • id

    • A unique id for each relationship

  • pid 1

    • The id of one of the entities the relationship is between

  • pid 2

    • The id of the other entity the relationship is between

  • relationship

    • The type of relationship

  • s_date

    • The beginning date of the relationship

  • end_date

    • The ending date of the relationship
